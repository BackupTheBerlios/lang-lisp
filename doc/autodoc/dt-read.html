The READ dictionary


[Function]
clause-add add &key stop => result


Produces an adding clause getting a symbol.


[Function]
clause-comment getstr &optional str end-comment-str => result


Zips past the comment. TODO C has escapes for these?


[Function]
clause-line-comment getstr &optional str => result


Zips past line comment. Currently assumes that things are read line by line.


[Function]
first-equal eql-to str => result


Sees if the first part of a string is equal to something.


[Function]
get-token str &optional stopchar => result


Gets a token from a string. Stopchar are characters it stops for, must are characters that must be in there, or it will also stop. Returns: the string token, the ending index, the element it stopped at.


[Function]
getstr-stream stream => result


A getstr for functions taking one.


[Function]
is-whitespace ch => result



[Function]
reader str getstr whitespace clauses &key limit => result


General reading, str is first string, getstr gets subsequent strings as needed, until it returns nil. Clauses is pairs of expected words and the functions get the string beyond, what these functions return is where reader continues.
