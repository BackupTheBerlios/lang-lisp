Lang development stages

Core development

The type system, resolving system, and manner of executing is developped. It is looked into whether this core can support programming methods and concepts with libraries. The goal is that the core should do as little as possible, and still be able to optimize, but be able to do as much as possible with libraries.

I will probably refrain from making optimalization mechanisms at this point, because then it will take longer to get into actually useable states.

Required milestones

More may still be added.
  • All working as should to create any library.
  • Conditionals, loops, functions, etc., macro creation from inside Lang.
  • A working 'Hello world', and a few working simple examples, at least one with (simple)graphics. Must test all the previous things.
  • Good unit tests. Here are unit tests to be past.

Libraries development

Beyond this point, less-then-core libraries, optimalization methods, and possibly conversion to more languages are developped. This can be done with more people more easily, since the libraries are largely separate.(They may aswel be separate projects.) Of course, it might need to be discussed how to keep the style consistent. A list of capabilities that should be in the language before going to the next step.(Again, i might add stuff.)

Required milestones

  • Simple or automatic usage of functions and structures from C.
  • Connections to many C libraries.
  • A loop, or iterate -like macro (preferably iterate-like)
  • Optimalization methods addable via libraries.
  • Mathematics and statistical functions, graphing functions.
  • A project to show it off a little.

Continued development, and promotion

When enough libraries for good use are developped, promoting the language for actual use can start. Meanwhile, there is no reason to stop making more libraries, improve optimization, and languages to convert to, and of course, fine tuning of existing code.
